Building a Ticketing System Using osTicket/Ubuntu/Apache/MySQL/PHP

Category : Web DevelopmentServer Administration

Ever since I joined the Middle Georgia Regional Library system, I had wanted to create a new ticketing system for everyone to use because Google Forms was lacking a lot of useful features that would make our jobs much easier. It began when I noticed that my department was spending a considerable amount of time communicating informationabout tickets before we had even begun the process of solving them. Questions such as: Have you seen that ticket from John yet?,Did you ever get a response?, and Help desk agents, when using Forms, are not able to directly respond to tickets, correspond with each other about tickets, produce statistics, mark tickets resolved, and many other things. Also, from time to time the Forms would get corrupted because of staff intervention. It did the job, but also came with a lot of problems, so I decided that the open-source and free osTicket system was a better option for the library. So, to create an osTicket system, I first needed a web server. There happened to be an unused server machine (a Dell PowerEdge T110 II) running Windows Server 2016, which would have been a contender to host the server using IIS. However, I felt that using Windows Server was overkill. There were too many features that werenâ€™t being used, and could stand to be an issue at a later point in time (e.g., Windows licenses). With this in mind, I decided to add an Ubuntu Linux partition to this server machine (still keeping the Windows Server partition intact). When booting the server, a user is greeted with a GRUB menu, letting them decide which OS they would like to use.
